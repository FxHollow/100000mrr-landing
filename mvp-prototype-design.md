**创建日期**: 2026-04-12
**负责人**: CEO

**版本**: v0.1

## 一、产品定位

**产品名称**: Emootions (暂定)

**Slogan**: 读懂情绪，预见关系

**目标用户**: 22-35 岁恋爱中/已婚人群

**核心价值**:
- 帮助用户识别和理解自己的情绪模式
- 通过 AI 分析预测关系中的潜在危机
- 提供科学的情绪调节和关系改善建议

## 三、用户流程

### 3.1 核心用户旅程

```
新用户注册
    ↓
产品引导 (价值说明 + 使用教程)
    ↓
首次情绪评估 (基线测量)
    ↓
每日情绪日记 → AI 分析 → 关系健康度更新
    ↓
收到预警/建议 → 执行调节练习
    ↓
定期回顾 (周报/月报)
```

### 3.2 情绪日记流程

```
用户打开 App
    ↓
点击「记录情绪」
    ↓
选择/输入当前情绪 (支持文字/语音)
    ↓
AI 追问引导 (可选)
  - 发生了什么？
  - 你的感受是？
  - 对方的反应是？
    ↓
提交记录
    ↓
AI 即时反馈
  - 情绪识别结果
  - 共情回应
  - 可选调节建议
```

### 3.3 关系预警流程

```
AI 持续分析用户数据
    ↓
检测到风险信号
  - 负面情绪频率上升
  - 互动质量下降
  - 特定关键词出现
    ↓
触发预警
  - 温和提醒 (非恐吓式)
  - 提供具体建议
  - 引导专业帮助 (如需要)
```

## 五、技术架构

### 5.1 前端

- **Web**: React + TypeScript
- **iOS**: Swift (原生) 或 React Native
- **Android**: Kotlin (原生) 或 React Native

### 5.2 后端

- **API**: Node.js + Express / Python + FastAPI
- **数据库**: PostgreSQL (用户数据) + MongoDB (日记内容)
- **AI 服务**: 自研 / 第三方 API (初步可用现有大模型)

### 5.3 数据隐私

- **端侧处理**: 敏感数据尽量在设备端处理
- **加密存储**: 所有用户数据加密存储
- **匿名化**: AI 训练数据脱敏处理

## 十、下一步行动

###  UI/UX 设计

**选项 1：外包设计公司**
- 预算：¥50,000-150,000
- 周期：2-4 周
- 优点：专业度高，交付稳定
- 缺点：成本较高，沟通成本

**选项 2：自由设计师**
- 预算：¥20,000-50,000
- 周期：3-5 周
- 优点：成本适中，灵活性高
- 缺点：需要寻找和筛选

**选项 3：内部设计（如有资源）**
- 预算：内部成本
- 周期：取决于资源
- 优点：沟通成本低，迭代快
- 缺点：需要内部有设计资源

### 开发团队

**选项 1：外包开发团队**
- 预算：¥150,000-300,000（MVP）
- 周期：8-12 周
- 优点：快速启动，专业交付
- 缺点：成本高，后期维护依赖

**选项 2：自建团队**
- 预算：人力成本（月薪¥25,000-40,000/人）
- 周期：招聘 4-6 周 + 开发 8-12 周
- 优点：长期可控，技术沉淀
- 缺点：前期投入大，招聘周期长

**选项 3：技术合伙人 + 外包**
- 预算：期权 + 外包费用
- 周期：取决于合伙人加入时间
- 优点：技术和产品可控，成本平衡
- 缺点：需要找到合适的合伙人

## 十一、建议方案

**推荐组合：外包设计 + 自建核心技术团队**

1. **UI/UX 设计外包**: 寻找有 SaaS 经验的设计师，预算¥30,000-50,000
2. **前端开发**: 招聘 1-2 名全职工程师，React/React Native 方向
3. **后端开发**: 招聘 1 名全职工程师，Node.js/Python 方向
4. **AI 服务**: 使用现有大模型 API + 自研微调

**理由**:
- MVP 阶段需要快速迭代，自建核心团队有利于长期发展
- 设计外包可以快速启动，减轻早期团队压力
- AI 能力是核心竞争力，需要自建团队掌握
