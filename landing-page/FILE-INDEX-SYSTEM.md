# 公司文件索引制度

> 生效日期：2026-03-28
> 维护角色：CEO
> 适用范围：所有公司成员（CEO/CMO/CTO 及所有 Agent）

---

## 一、制度目的

董事长反馈：人一多分不清各个角色的产出都放在哪了。

为解决此问题，特制定本制度：**所有角色今后有任何产出，必须备注文件地址**。

---

## 二、文件归档原则

### 2.1 基本原则

1. **谁产生，谁归档** - 文件创作者负责在正确位置保存并记录
2. **即创即记** - 创建文件的同时在索引中登记
3. **路径清晰** - 使用有意义的路径和文件名
4. **版本可溯** - 重要文档通过 Git 管理版本

### 2.2 文件命名规范

```
[日期]-[类型]-[描述].md
```

示例：
- `2026-03-28-marketing-weekly-report.md`
- `2026-03-27-api-user-authentication.md`
- `2026-03-26-customer-interview-notes.md`

---

## 三、各类产出归档要求

### 3.1 代码提交

| 项目 | 要求 |
|------|------|
| **仓库路径** | 必须标明所属仓库（如 `100000mrr-landing/backend`） |
| **Commit Hash** | 重要提交记录 commit hash |
| **说明** | 在 commit message 中说明变更目的 |

**示例记录：**
```md
- 新增用户认证 API: `backend/src/routes/auth.js` (commit: 36b9e09)
```

### 3.2 文档产出

| 类型 | 存储位置 | 记录要求 |
|------|----------|----------|
| 内部文档 | 对应角色目录 `/cmo/`, `/cto/`, `/ceo/` | 文件路径 + 简要说明 |
| 对外文档 | `content-articles/` 或 `marketing-assets/` | 文件路径 + 发布状态 |
| 会议纪要 | `daily-notes/` 或对应项目目录 | 日期 + 参会人 + 决策项 |

### 3.3 报告生成

| 报告类型 | 存储位置 | 记录要求 |
|----------|----------|----------|
| 日报 | `daily-notes/YYYY-MM-DD.md` | 日期 + 完成事项 |
| 周报 | 对应角色目录 `weekly-report-*.md` | 周期 + 关键指标 |
| 专项报告 | 对应项目目录 | 主题 + 结论摘要 |

### 3.4 数据文件

| 类型 | 存储位置 | 记录要求 |
|------|----------|----------|
| CSV/Excel | `marketing-assets/` 或 `backend/data/` | 路径 + 数据说明 + 更新频率 |
| 配置文件 | 根目录或对应服务目录 | 路径 + 用途说明 |
| 数据库 | `backend/prisma/` | Schema 路径 + 迁移记录 |

- **CEO 审核**：每周检查索引完整性

## 六、Git 提交规范

### 6.1 Commit Message 格式

```
<type>: <subject>

[optional body]

Co-Authored-By: Paperclip <noreply@paperclip.ing>
```

### 6.2 Type 说明

| Type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档更新 |
| `style` | 格式调整 |
| `refactor` | 重构 |
| `test` | 测试相关 |
| `chore` | 构建/工具/配置 |
| `update` | 进度/状态更新 |

### 6.3 示例

```bash
git commit -m "docs: 新增文件索引制度文档"
git commit -m "update: CMO README - 更新 git 提交历史"
git commit -m "feat: 添加用户认证 API"
```

## 七、飞书/云端文档同步

如使用飞书等云端文档工具：

1. **双向链接**：本地索引文件应包含飞书链接
2. **定期备份**：重要云端文档应导出本地备份
3. **权限管理**：确保团队成员有访问权限

---

## 八、违规处理

1. **首次发现**：提醒补充索引
2. **多次发现**：在周报中通报
3. **持续改进**：CEO 定期优化制度流程
